ZiLogger: Comment l'utiliser ?

ven. 25 janvier 2019

ZiLogger

Fonctionnement:

Afin de faciliter le débug de vos ZiA, nous avons mis en place un système de log.
Notre logger fonctionne à l'aide de sévérité et de type d'erreur.

Sévérité:

La sévérité définie si votre log doit être afficher ou non. C'est un paramètre que vous choisissez au début de votre Zia. La sévérité est définie comme suit:

 enum class Severity : int {
            USELESS = 0,
            NORMAL,
            IMPORTANT,
            VITAL
        };

Ainsi, si vous décider de n'afficher que les sévérités NORMAL et supérieures, les log définis comme USELESS ne s'afficheront pas. Les sévérités VITAL ne peuvent pas être enlevées.

Type de log:

Le type de log défini, le balise qui se trouvera devant votre log.
Les différents types sont définis comme suit:

 enum class Type {
            INFO,
            DEBUG,
            WARNING,
            ERROR,
            FATAL,
            ABORT_MISSION
        };

Par exemple si l'on décide d'afficher un log de type ERROR, la balise serait "[ERROR]:".
Petite particularité pour ABORD_MISSION, ce type de log exit votre ZIA, à n'utiliser qu'en cas d'erreur très importante.

Output:

Notre système de log vous permet de choisir l'output de chacun de vos log.
Les outputs sont définis comme suit:

 enum class OutputStream {
            COUT,
            CERR,
            FILE,
        };

Comme vous l'avez surement deviné, COUT pour la sortie standard, CERR pour la sortie d'erreur, et FILE pour écrire dans un fichier.

Vous pouvez facilement changer le fichier dans lequel vous voulez que la sortie FILE écrive en utilisant la fonction suivante:

static inline void setStreamFile(const std::string &filePath = "./log.txt");

Macro Log()

La macro Log() permet d'afficher la ligne et le fichier duquel sort le log. Cette macro est définie comme suit:

#define Log(logType) logType << __FILE__ << ":" << __LINE__ << " : "

Elle ne fonctione que sur un système unix.
